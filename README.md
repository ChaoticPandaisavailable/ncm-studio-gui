# NCM Studio GUI

这是给 `ncmdump.exe` 做的本地图形前端。

## 启动

双击：

```text
start_ncmdump_gui.bat
```

GUI 会自动查找：

```text
D:\ncm2mp3\ncmdump-1.5.1-windows-amd64\ncmdump.exe
```

如果移动过文件，可以在界面里的“转换核心”处手动选择 `ncmdump.exe`。

## 使用

1. 点击“添加文件”选择一个或多个 `.ncm`。
2. 或点击“添加文件夹”批量转换目录里的 `.ncm`。
3. 选择输出目录；留空则输出到源文件旁边。
4. 点击“开始转换”。

## 选项

- “递归文件夹”：扫描所选文件夹的所有子文件夹。指定输出目录时，会保留原来的子目录结构。
- “跳过已转换”：如果输出目录里已经有同名 `.mp3` 或 `.flac`，就不再转换该 `.ncm`。
- “扫描”：立即扫描队列，并在日志里列出待转换和会被跳过的已转换文件。
- “成功后删除源文件”：转换成功后删除原始 `.ncm`，请谨慎使用。
- 日志会先列出“待转换清单”，再列出“已转换跳过”，方便先看到真正还需要处理的文件。

## 说明

`ncmdump.exe` 本身是命令行工具，直接双击会很快退出，看起来像闪退。这个 GUI 会负责选择文件、拼接参数、显示日志，并隐藏命令行窗口。

“开始转换”“停止”“扫描”“打开输出目录”和进度条已经固定到上方操作带，窗口高度变小时不会再被右侧栏裁掉。

GUI 会把上次的输入队列、输出目录、转换核心路径和开关状态保存到：

```text
ncmdump_gui_state.json
```

下次启动时会自动恢复。
